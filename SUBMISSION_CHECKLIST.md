## Submission Methods

### Method 1: ZIP File (Recommended for first submission)
- Ensure file size is under 25MB (should be ~500KB without node_modules/venv)
- Name format: `[YourName]-Loadsmart-Assessment.zip`

### Method 2: Private GitHub Repository
- Create a new private repository
- Add recruiter's GitHub username as collaborator
- Send repository link via email

### Method 3: Google Drive
- Upload ZIP file to Google Drive
- Set sharing permissions to "Anyone with the link"
- Send link via email

## Common Pitfalls to Avoid

- [ ] Don't include `node_modules/` or `venv/` in submission
- [ ] Don't include `.env` files with actual credentials
- [ ] Don't make repository public (per instructions)
- [ ] Don't include compiled files (`__pycache__`, `dist/`, etc.)
- [ ] Don't leave debugging code or console.logs
- [ ] Don't use generic commit messages like "fix" or "update"
- [ ] Don't reference AI tools in commits or code
